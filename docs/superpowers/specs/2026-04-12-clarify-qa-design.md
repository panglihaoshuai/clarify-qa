# Clarify Q&A — 产品设计规格文档

## 概述

**产品名称：** Clarify Q&A
**版本：** 1.0
**日期：** 2026-04-12
**状态：** 设计阶段

---

## 一、产品定位

**一句话描述：** 独立部署的 AI 问答工具，帮助员工将模糊指令拆解清晰，并支持持续对话追问。

**核心用户：** 小团队（10人以内的部门），普通办公用户

**核心价值：** 把模糊的想法变得清晰、可执行

---

## 二、用户流程

```
上传文档/图片（最多20个，拖拽）
    ↓
选择分析方法论（标签按钮）
    ↓
生成结构化分析报告
    ↓
查看报告 + 继续对话追问
```

---

## 三、功能模块

### 3.1 账号系统

- 邮箱 + 密码注册
- 初始密码：123456（不可修改）
- 数据隔离：用户只看自己的分析记录

### 3.2 文件上传

- 拖拽上传或点击选择
- 最多 20 个文件（文档 + 图片混合）
- 支持格式：PDF、图片（jpg/png/gif/webp）、Word、txt
- 图片：直接传给 MiniMax，单张 ≤ 5MB
- 文本文件：前端自动分片，控制 token 预算

### 3.3 方法论选择

预装 5 个 Skill（用户可选一个，黑名单机制）：

| Skill 名称 | 来源 |
|-----------|------|
| brainstorming | superpowers 内置 |
| 马斯克 | `github.com/alchaincyf/elon-musk-skill` |
| 乔布斯 | `github.com/alchaincyf/steve-jobs-skill` |
| Paul Graham | `github.com/alchaincyf/paul-graham-skill` |
| 芒格 | `github.com/alchaincyf/munger-skill` |

**交互：** 标签/药丸按钮，默认选中"通用分析"，用户可切换

### 3.4 分析报告

生成两个维度：

1. **模糊点识别** — 指出指令里哪里表述不清晰、可能产生歧义
2. **可执行项建议** — 拆解成具体可执行的子任务

### 3.5 追问对话

- 报告生成后保留对话入口
- 用户可以继续多轮追问
- 上下文保持，AI 记住之前的分析内容

### 3.6 历史会话

- 保存所有分析会话
- 可回看历史报告
- 可继续之前的对话
- 可删除会话

## 五、技术架构

### 5.1 技术栈

| 层级 | 技术选择 |
|------|---------|
| 前端 | Next.js（App Router） |
| 后端 | Next.js API Routes |
| 数据库 | PostgreSQL on Railway |
| AI | MiniMax API（直接调用） |
| 文件处理 | 前端分片，直接调 API，零存储 |
| 部署 | Railway（独立部署，可嵌入任意网站） |

### 5.2 数据库模型

```sql
User（用户）
├── id (uuid, PK)
├── email (唯一)
├── password_hash
├── created_at

Session（会话）
├── id (uuid, PK)
├── user_id (FK → User)
├── title (自动生成)
├── methodology (选用的方法论)
├── status (processing / completed)
├── created_at
├── updated_at

Message（消息）
├── id (uuid, PK)
├── session_id (FK → Session)
├── role (user / assistant)
├── content (文本)
├── created_at

FileMeta（文件元数据，仅存引用）
├── id (uuid, PK)
├── session_id (FK → Session)
├── file_name
├── file_type
├── file_size
├── mini_max_ref
├── created_at
```

### 5.3 API 设计

```
POST /api/auth/register     # 注册
POST /api/auth/login        # 登录

GET  /api/sessions              # 获取用户所有会话
POST /api/sessions              # 创建新会话
GET  /api/sessions/:id          # 获取会话详情（含消息）
DELETE /api/sessions/:id        # 删除会话

POST /api/sessions/:id/messages  # 发送消息（首次触发分析）
GET  /api/sessions/:id/messages  # 获取会话消息历史
```

## 六、文件分片策略

### 6.1 分片规则

- **图片**：直接传 MiniMax，最多 20 张，单张 ≤ 5MB
- **文本文件**：
  - 总 token 预算：MiniMax 单次上下文上限 × 0.8（留余量）
  - 超出时自动分批，每批独立分析
  - 最后合并各批结论，去重 + 建立关联，生成统一报告

### 6.2 用户体验

对用户完全透明，不需要手动分组。报告生成时显示处理进度。

## 七、MiniMax 集成方案

### 7.1 MCP 模式

用户已有 MiniMax MCP 配置，工具直接调用本地 MCP 工具。

### 7.2 Skill 注入

每个 Skill 对应一个**提示词模板**，存在数据库或配置文件中。用户选择方法论 → 系统将该方法论的提示词模板注入到 MiniMax 的 system prompt 中。

### 7.3 MiniMax 能力使用

- **图片理解**：分析上传的截图、文档照片
- **网络搜索**：补充实时信息（可选，视 API 能力而定）

## 八、Skill 集成详情

### 8.1 预装 Skill 清单（1.0）

| Skill | 来源 | 用途 |
|-------|------|------|
| brainstorming | superpowers 内置 | 通用思维整理 |
| 马斯克 | `alchaincyf/elon-musk-skill` | 第一性原理分析 |
| 乔布斯 | `alchaincyf/steve-jobs-skill` | 产品/设计思维 |
| Paul Graham | `alchaincyf/paul-graham-skill` | 创业/写作思维 |
| 芒格 | `alchaincyf/munger-skill` | 多元心智模型 |

### 8.2 黑名单机制

- 默认只预装上述 5 个 Skill
- 其他 Skill 被屏蔽（黑名单）
- 用户在提问时可指定使用其中某个方法论

### 8.3 Skill 加载方式

每个 Skill 的 `SKILL.md` 文件内容作为提示词模板，在调用 MiniMax 时注入到 system prompt。
